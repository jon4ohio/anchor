# The Problem

Projects accumulate knowledge. Conversations, docs, ADRs, AI context files, architecture notes — spread across tools and sessions.

When you return after a week, a month, or a new AI session, you reconstruct context instead of continuing work.

---

## The transformation

**Continue instead of reconstruct.**

| Before | After |
|--------|-------|
| Every session starts with "Here's my project…" | Work continues from durable context |
| Decisions live in chat history | Decisions stay discoverable |
| Returning means archaeology | Returning means reading Handoff and owners |
| Context accumulates accidentally | Context accumulates intentionally |

---

## Any project

Anchor brings durable coordination to **any project** — one day old or five years old.

Anchor starts by understanding your project as it exists today. It does not ask you to throw away ADRs, `CLAUDE.md`, or architecture docs that already work.

---

## The core idea

**Knowledge has responsibilities.**

Decisions, identity, session state, and expertise are different jobs. When one file does three jobs, humans and AI lose trust in what to read first.

**Contracts formalize responsibilities.** They name owners so everyone knows where each kind of truth lives.
