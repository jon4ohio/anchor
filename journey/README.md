# Reference Projects

Reference Projects are Anchor's **proving ground** — where theory meets real software.

They are not framework contracts. They combine evidence, demonstration, teaching, and validation in one layer.

---

## The bridge

```text
Experience + Practices
        │
        ▼
Reference Projects     ← you are here
        │
        ▼
Independent Adoption
        │
        ▼
Evidence
        │
        ▼
Framework Evolution
```

Reference Projects let adopters see practices working before they commit. They let maintainers collect evidence before promoting framework changes.

## How to use Reference Projects

**Newcomers:** Start with [john-ohio](john-ohio/README.md) — read [NARRATIVE.md](john-ohio/NARRATIVE.md) to see adoption before doing it yourself.

**Adopters:** Compare your repository structure to each session snapshot. Ask: "Which responsibility does this file have?"

**Contributors:** Record observations from independent use in [evidence.md](../releases/v0.2/evidence.md). See [Contributing Evidence](../docs/practices/contributing-evidence.md).

**Maintainers:** Reference Projects inform Evidence Review — they do not replace independent adopter evidence.
